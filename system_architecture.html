DuckDB 多线程分析型引擎。解耦策略，利用窗口函数与 JOIN 在 6秒左右 运算全市场千万行因子。

⚙️ 三、 核心模块深度解剖 (Core Modules)

01 数据解析与前复权计算层
parser/
本层是量化系统的“数字前线”，负责剥离和解算通达信的物理混淆，提供纯净的结构化数据。
  • K线解析 (kline.py)：使用 NumPy `fromfile` 搭配 32 字节定长 C 结构体映射，彻底规避 Python 的循环开销。
  • GBBQ解密 (gbbq.py)：采用 16 轮循环 XOR 异或密码网络，破解出除权除息及送转明文数据。
  • 前复权计算 (adjuster.py)：自后向前修正价格，且同步修正成交量(Volume)保证 Close * Volume = Amount 金融守恒。
  • 板块解算 (block.py)：精准破解概念、风格及指数板块的 2813 字节定长块结构，输出多维映射表。
02 高持久化列式存储层
storage/
解决高频大体量回测的持久化和读写瓶颈，是系统的本地高性能数据缓存。
  • 物理文件隔离 (parquet_store.py)：单股独立物理文件，支持仅读取所需时间序列。
  • Snappy 列投影加速：由于采用了高性能的 Apache Parquet 格式，选股时仅需投影 Date 和 Close 列，I/O 吞吐提升 10 倍以上。
  • 板块对照表：将全市场 7.3 万条个股与板块的映射以 Parquet 形式固化，支持毫秒级关联查询。
03 多进程增量同步引擎
sync_market.py
量化数据新鲜度的保障引擎。通过底层极速调度，秒级一键同步全市场标的。
  • 增量 mtime 毫秒级比对：通过秒级比对源文件与 Parquet 的修改时间，只同步有变动的标的，全同步时间从数分钟暴省至20秒。
  • Fork-Safe 共享内存优化：子进程初始化器直接读取预解密 GBBQ DataFrame 的内存地址，消除了数十万条数据在进程间的序列化/进程通信开销。
04 解耦式极速 SQL 选股引擎
screener.py & strategies.json
整个量化系统的决策大脑，负责以多线程高并发状态在秒级对全市场进行因子分析与战术股筛选。
  • 代码与算法配置分离：策略因子公式与 SQL 计算模板完全存放在 strategies.json 中，新增或修改策略完全无需改动 Python 源码。
  • 自下而上资金宽度策略 (Sector Breadth)：SQL 窗口函数极速在内存中统计并计算当天站上 MA20 且温和放量突破的个股占所属概念板块的总比重，精准剔除无板块合力的冷门庄股。
  • 报告物理隔离：各策略独立输出 `{strategy}_report.md` 报告，完全互不干扰。
